# Continue Watching (cineby + rivestream)

A browser extension that adds a **Continue Watching** list and **resume-from-last-position**
to free streaming sites that don't have it — currently **cineby** and **rivestream**.

- Tracks your position in each movie/episode as you watch.
- Shows a **"Resume from MM:SS"** prompt when you reopen something you didn't finish.
- **In-page panel:** a "Continue Watching" panel appears on the site (and auto-opens when
  you land on the home/browse pages) with poster, title, progress bar, last-watched.
- **Toolbar popup:** click the extension icon for the same list in a clean popup.
- **Per-device** storage — no account, no backend, no tracking. Nothing leaves your browser.
- **Export / Import JSON** for manual backup or moving history between devices.

> These sites have no accounts or API, so the feature runs *inside the page* via a content
> script. There's no server and nothing to sign into.

## Install on your Mac (Chrome)

1. Open Chrome → go to **chrome://extensions**.
2. Turn on **Developer mode** (toggle, top-right).
3. Click **Load unpacked**.
4. Select the **`extension`** folder inside this project:
   `/Users/ankushgochke/Documents/Workspace/video-ext/extension`
5. The extension appears with a red ▶ icon. Click the puzzle-piece 🧩 in the toolbar and
   **pin** it so the icon is always visible.

That's it — no Tampermonkey, no paste step. (Works the same in Edge/Brave: `edge://extensions`,
same steps.)

### Updating
Edit files in `extension/`, then click the **↻ reload** icon on the extension's card in
`chrome://extensions`. Your history is preserved across reloads.

1. Go to **https://www.cineby.at/movie/758330** and press play.
2. Watch ~1 minute (it starts saving after the 30-second mark).
3. Click the toolbar ▶ icon — the movie shows with a progress bar. The in-page ▶ button

## How it works (mental model)

- A **content script** runs only on cineby/rivestream pages.
- It reads a stable content id from the URL (e.g. `/movie/758330` → `cineby:movie:758330`,
  `/tv/1399/1/1` → `cineby:tv:1399:1:1`).
- It waits for the `<video>` element, then saves your position every ~5s while playing,
  plus on pause / tab-hide / close.
- Items past **95%** are treated as finished and drop off automatically. Things opened for
  <30s aren't recorded.
- Everything is stored via `chrome.storage.local` (per device).

## cineby specifics (already checked)

cineby is a **Next.js** SPA, routes `/movie/[...params]` and `/tv/[...params]`. Confirmed
the URL → content-id parsing works, so **tracking and resume work out of the box**.

The HTML meta tags are **not** usable for titles/posters — the shell ships generic
defaults (`og:title` = `"Cineby"`, `og:image` = a generic `seo.png`) and the real movie
title/poster are injected by React from TMDB after hydration. The adapter handles this: it
ignores the generic values, re-reads title/poster on every save, and shows a fallback label
like **"Movie 758330"** until the real title appears, then upgrades it.

**If the panel keeps showing "Movie 758330"** instead of the real name, open the DevTools
console on a loaded watch page and run:

```js
document.title
[...document.querySelectorAll('h1,h2')].map(e => e.textContent).filter(Boolean)
document.querySelector('img[src*="image.tmdb.org"]')?.src
```

Paste whichever holds the real title into `extractTitle()` (and the working image selector
into `extractPoster()`) in the cineby adapter in `content.js`. Core tracking/resume does not
depend on this — it's display polish.

## Adding another site

In `content.js`, extend the adapters list:

```js
const ADAPTERS = [
  { test: (h) => /(^|\.)cineby\./i.test(h),     adapter: makeTmdbAdapter('cineby') },
  { test: (h) => /(^|\.)rivestream\./i.test(h), adapter: makeTmdbAdapter('rivestream') },
  // { test: (h) => /(^|\.)yoursite\./i.test(h), adapter: makeTmdbAdapter('yoursite') },
];
```

Then add a matching entry to `content_scripts[0].matches` in `manifest.json` and reload the
extension. If the site uses a different URL scheme, write a custom adapter object with the
same methods (`matches`, `extractContentId`, `extractType`, `extractTitle`, `extractPoster`,
`getVideo`) instead of `makeTmdbAdapter`.

> Sites that play video inside a **cross-origin iframe** (vidsrc/embed-style) need extra
> work (a postMessage bridge) and aren't supported yet. cineby and rivestream both use a
> native `<video>` in the page, which keeps this simple.

## Cross-device sync (optional, end-to-end encrypted)

Your watch list can sync across devices/browsers — including a future Firefox phone — via a
tiny Cloudflare Worker you own. **Your passphrase encrypts the data on-device (AES-GCM)
before it's uploaded**, so Cloudflare only ever stores ciphertext and never sees your
passphrase. The server key is a SHA-256 hash of the passphrase, so the server can't even
tell which data is yours.

**Deployed worker URL:** `https://cw-sync.ankushgochke.workers.dev`

### Turn on sync in the extension

1. Click the extension's ▶ icon → the **⚙ gear** (top-right of the popup).
2. Paste the **worker URL** `https://cw-sync.ankushgochke.workers.dev` and pick a
   **passphrase** (anything memorable — this is the key, so use the *same* one on every
   device).
3. Click **Save & sync**. Chrome will ask permission to reach your worker — allow it.
4. Status shows "Synced just now". From then on it auto-syncs on changes, every 5 minutes,
   and on browser startup. **Sync now** forces it.

To sync another device, install the extension (or the userscript) there, open the same
settings, and enter the **same worker URL + same passphrase**. Lists merge automatically,
keeping the most-recent position for each title.

> Need to redeploy the worker, or set it up somewhere else? The deploy steps live in
> [`sync-worker/README.md`](sync-worker/README.md).

The phone **userscript has the same sync** built in (gear ⚙ inside its panel). It uses the
identical encryption, so the extension and userscript share one synced list when given the
same worker URL + passphrase. Sync timing model: writes happen only when your progress
actually changes (keeps you under the free KV write limit); reads (pulls) happen when you
open the site, on window focus, and on a 1-minute timer — so opening cineby on any device
refreshes its list within ~1 second.

> The userscript (for the phone) doesn't have the sync UI wired up yet — that's the next
> step once the extension sync is confirmed working. Same worker + passphrase will apply.

## Android

Chrome on Android **cannot** run extensions or userscripts at all. Use **Firefox for
Android**, which runs the userscript (`continue-watching.user.js`) — it has the full feature
set including the same encrypted sync.

### Steps

1. Install **Firefox** for Android.
2. Install **Tampermonkey** (or Violentmonkey): Firefox menu → **Add-ons** → search
   Tampermonkey → Add.
3. In Firefox, open the userscript's Raw URL — Tampermonkey detects it and offers
   **Install** (this URL also auto-updates the script via `@updateURL`):
   ```
   https://raw.githubusercontent.com/flank2296/continue-watching/main/continue-watching.user.js
   ```
4. Open **cineby** in Firefox, then the ▶ panel → **⚙** → enter the worker URL
   `https://cw-sync.ankushgochke.workers.dev` and the **same passphrase** as your Mac →
   **Save & sync**. Your list now matches across both devices.

> Alternative: **Kiwi Browser** (Chromium) can load the unpacked `extension/` directly, but
> Kiwi is no longer maintained — Firefox + userscript is the more durable path.

## Privacy

All data stays on the device in the browser's extension storage. There are no network
requests, no analytics, no sync. Use **Export** for a backup or to copy your list to another
device manually.
